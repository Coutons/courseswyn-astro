It's the architectural backbone of modern agentic AI — the layer that separates chatbots from agents that <strong class="text-[hsl(222,47%,8%)]">actually do things</strong>: browse the web, query databases, write files, trigger APIs, automate workflows, and more.</p>

  <h2 class="text-2xl font-black text-[hsl(222,47%,8%)]">MCP Developer Glossary & Core Concepts</h2>
  <p class="text-[hsl(215,20%,45%)] text-sm md:text-base leading-relaxed">If you're entirely new to the Model Context Protocol ecosystem, understanding these core technical terms will accelerate how fast you can build agentic AI integrations.</p>
  
  <div class="grid grid-cols-1 md:grid-cols-2 gap-x-8 gap-y-6 mt-4">
    <div class="space-y-2">
      <h3 class="font-bold text-[hsl(222,47%,8%)] text-lg border-b border-[hsl(0,0%,92%)] pb-2">MCP Host</h3>
      <p class="text-[hsl(215,20%,45%)] text-sm leading-relaxed">The application that humans interact with. Examples include Claude Desktop, <a href="/blog/best-cursor-ai-courses-udemy/" class="text-[hsl(215,25%,35%)] hover:text-[hsl(215,70%,55%)] underline decoration-blue-500/50 underline-offset-2">Cursor IDE</a>, or a custom Streamlit frontend. The host initiates the connection to MCP clients.</p>
    </div>
    <div class="space-y-2">
      <h3 class="font-bold text-[hsl(222,47%,8%)] text-lg border-b border-[hsl(0,0%,92%)] pb-2">MCP Client</h3>
      <p class="text-[hsl(215,20%,45%)] text-sm leading-relaxed">The component within an application that maintains 1:1 connections with one or more MCP servers. The client routes requests from the host architecture to the appropriate server logic.</p>
    </div>
    <div class="space-y-2">
      <h3 class="font-bold text-[hsl(222,47%,8%)] text-lg border-b border-[hsl(0,0%,92%)] pb-2">MCP Server</h3>
      <p class="text-[hsl(215,20%,45%)] text-sm leading-relaxed">A lightweight backend program (often written in TypeScript or Python) that exposes specific local or external capabilities (like database access or API execution) back to the client.</p>
    </div>
    <div class="space-y-2">
      <h3 class="font-bold text-[hsl(222,47%,8%)] text-lg border-b border-[hsl(0,0%,92%)] pb-2">Tools (Tool Calling)</h3>
      <p class="text-[hsl(215,20%,45%)] text-sm leading-relaxed">Executable functions exposed by an MCP Server. For example, a "Local File Search" tool that an LLM can request the server to execute on its behalf to retrieve necessary context.</p>
    </div>
    <div class="space-y-2">
      <h3 class="font-bold text-[hsl(222,47%,8%)] text-lg border-b border-[hsl(0,0%,92%)] pb-2">Resources</h3>
      <p class="text-[hsl(215,20%,45%)] text-sm leading-relaxed">Static or dynamic read-only data exposed by an MCP Server, similar to a file system or API endpoint. These are injected directly into the LLM context prompt.</p>
    </div>
    <div class="space-y-2">
      <h3 class="font-bold text-[hsl(222,47%,8%)] text-lg border-b border-[hsl(0,0%,92%)] pb-2">Stdio vs. SSE Connections</h3>
      <p class="text-[hsl(215,20%,45%)] text-sm leading-relaxed">The transport layers MCP uses. `stdio` is for local execution (client and server on the same machine), while `SSE` (Server-Sent Events) is for remote server connections over HTTP.</p>
    </div>
